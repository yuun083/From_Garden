# Прямо с грядки - Фермерские продукты

## Описание проекта

"Прямо с грядки" - это веб-приложение для покупки фермерских продуктов напрямую у производителей. Проект позволяет пользователям находить и заказывать свежие фермерские продукты, а также позволяет фермерам продавать свою продукцию.

## Функциональные возможности

### Для пользователей:
- Просмотр каталога продуктов
- Поиск и фильтрация продуктов по категориям
- Просмотр информации о фермерах
- Добавление продуктов в корзину
- Оформление заказов
- Подписка на регулярные поставки
- Оставление отзывов
- Управление профилем

### Для фермеров:
- Регистрация и подтверждение фермы
- Управление ассортиментом продукции
- Просмотр заказов и отзывов
- Управление информацией о ферме

### Для администраторов:
- Управление пользователями
- Управление продуктами
- Управление фермами
- Модерация отзывов
- Управление категориями и единицами измерения

## Установка

1. Клонируйте репозиторий:
```bash
git clone <url_репозитория>
cd <имя_папки_проекта>
```

2. Создайте виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate  # На Linux/Mac
# или
venv\Scripts\activate  # На Windows
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```


5. Запустите инициализацию базы данных:
```bash
python init_db.py
```

6. Запустите приложение:
```bash
python main.py
```

## Настройка

### Конфигурация базы данных

В файле `.env` укажите следующие параметры:

- `SECRET_KEY` - секретный ключ для JWT токенов
- `ALGORITHM` - алгоритм шифрования для JWT токенов
- `ACCESS_TOKEN_EXPIRE_MINUTES` - время жизни токена в минутах
- `DB_NAME` - имя файла базы данных (для SQLite) или название базы данных

## API документация

После запуска приложения документация к API доступна по адресу:
- `/api/docs` - интерактивная документация Swagger
- `/api/redoc` - документация ReDoc

## Структура проекта

```
app/
├── api/          # Роутеры FastAPI
├── models/       # Модели SQLAlchemy
├── repositories/ # Репозитории для работы с базой данных
├── services/     # Бизнес-логика
├── schemes/      # Pydantic схемы
├── exceptions/   # Исключения
├── database/     # Конфигурация базы данных
├── static/       # Статические файлы
├── templates/    # Шаблоны Jinja2
└── config.py     # Конфигурация приложения
```
